Class env_t

source code

object --+
         |
        env_t

The is the environment for compilation. The environment includes a dictionary for holding everything custom about this environment. The default environment includes:

  • command line options. These are also in the environment dictionary.
    • build_dir default: obj
    • src_dir default: . or path to the mfile
    • gen_dir default: None (default path for generated files, if set)
    • shared default: False (default: no shared libraries)
    • static default: False (default: not to statically link)
    • opt default: 'noopt' (could be 'noopt, 0,1,2,3)
    • debug default: False
    • separate_pdb_files default: False
    • targets default: [] targets to build
    • verbose default: 1
    • compiler default: 'gnu', 'ms', 'clang', 'icc', 'icl', 'iclang'
    • extra_defines default: ''
    • extra_flags default: '' (for both CXXFLAGS & CCFLAGS)
    • extra_cxxflags default: ''
    • extra_ccflags default: ''
    • extra_linkflags default: ''
    • extra_libs default: ''
    • use_yasm default: False
  • CPPPATH default: [] The list of include paths
  • SYSTEMINCLUDE default: [] The list of system include paths (Not supported by MSVS).
  • DEFINES default: {} The dictionary of defines
  • short names for the primary compiler tools:
    • CXX_COMPILER cl or g++
    • CC_COMPILER cl or gcc
    • ASSEMBLER ml/ml64 or gcc/gas (gcc is the default for gnu)
    • LINKER link or g++/gcc (g++ is the default for gnu)
    • ARCHIVER ar
    • RANLIB_CMD ranlib
  • toolchain path to the compiler tools (default is ''). If toolchain is set, it should end with a trailing slash.
  • vc_dir path to the compiler VC directory for MSVS (default is '')n
  • icc_version 7, 8, 9, 10, ...
  • gcc_version 2.96, 3.x.y, 4.x.y, ...
  • msvs_version 6 (VC98), 7 (.NET 2003), 8 (Pro 2005), ...
  • primary compilation tools:
    • CC cl or gcc (with toolchain path)
    • CXX cl or g++ (with toolchain path)
    • AS ml,ml64 or gcc/gas (with toolchain path)
    • LINK link or gcc/g++ (with toolchain path)
    • AR lib or ar (with toolchain path)
    • RANLIB ranlib (with toolchain path)
  • flags for primary tools:
    • CCFLAGS
    • CXXFLAGS
    • ASFLAGS
    • ARFLAGS
    • LINKFLAGS
    • LIBS (libraries for the end of the link statement)
  • preprocessor flags
    • DOPT /D or -D
    • ASDOPT /D or -D
    • IOPT /I or -I
    • OPTPOPT /O or -O
    • DEBUGOPT /Zi or -g
  • options to control compilation output:
    • COPT /c or -c
    • COUT /Fo or -o
    • ASMOUT /Fo or -o
    • LIBOUT /out: or -o
    • LINKOUT /OUT: or -o
    • DLLOPT -shared
  • Override-builders: set these to a function pointer if you want to replace the default builder function.
    • ASSEMBLE_BUILDER if not set default is to use assemble_default()
    • CXX_COMPILE_BUILDER if not set default is to use cxx_default()
    • CC_COMPILE_BUILDER if not set default is to use cc_default()
    • LINK_BUILDER if not set default is to use link_default()
    • STATIC_LIBRARY_BUILDER if not set default is to use static_lib_default()
    • DYNAMIC_LIBRARY_BUILDER if not set default is to use dynamic_lib_default()
  • default extensions:
    • OBJEXT .obj or .o
    • LIBEXT .lib or .a
    • DLLEXT .dll, .so, or .dylib
    • EXEEXT .exe or ''
  • System values:
    • uname standard python tuple of values from uname.
    • system standard values: 'Linux', 'Windows', 'Darwin', 'Microsoft', 'FreeBSD'
    • hostname
    • build_os standard values: 'lin', 'win', 'mac', 'bsd'
    • host_os standard values: 'lin', 'win', 'mac', 'bsd'
    • build_cpu standard values: 'ia32', 'x86-64', 'ipf'
    • host_cpu standard values: 'ia32', 'x86-64', 'ipf'

add_to_var(self, var, value)

source code 

Add or append value to the environment variable var. If the variable is not in the environment, then it is added as is. Otherwise if the variable is in the environment and is a list then value is appended. Otherwise, the value is appended as a string with a leading space. This will *NOT* do variable substitution when adding to a variable.

Parameters:
  • var (string) - name of a dictionary key
  • value (string) - the value to add or append
